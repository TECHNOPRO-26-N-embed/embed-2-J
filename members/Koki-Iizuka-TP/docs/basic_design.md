## 1. システム全体像

### 1-1. ブロック構成（入力 → 処理 → 出力）

```
【入力】                                  【Arduino UNO R3】                 【出力】
タクトスイッチ①（再生/停止ボタン） ──→┐
                                         │
タクトスイッチ②（曲切替ボタン）   ───→┤    状態管理・判定処理    ├───→ LED
                                         │                            ├──→ パッシブブザー
                                         │                            └──→ LCDディスプレイ

補足説明（図で表現しきれない場合）
- 「タクトスイッチ①：曲の再生／停止を操作」
- 「タクトスイッチ②：再生中または待機中に曲を切り替える（複数曲対応時）」
- **曲再生時、自動的に「次の曲」へ進む（最後の曲で自動停止）**
```
---

### 1-2. 状態遷移（ガジェットの「ふるまい」設計）

> ※ ガジェットが「どんな状態を持つか」を定義します。
> この設計が loop() の骨格になります。

**↓ 状態遷移修正版（自動次曲/自動停止あり）**

```
[電源ON/初期化]
      ↓
  [待機中]  ←────────┐
   |  ↑                 |
   |  └──(再生/停止)    |
   |                    |
(再生/停止押下)          ↓
   ↓              [再生中]
  [再生中]
   |  ↑    ┌──────────────┐
   |  |    | 曲途中で停止  |
   |  └───(再生/停止押下)↓
   |                [待機中]
   |
   └▷（曲が終わる）
          ├─［次曲あり］────────→ [再生中]（次の曲へ）
          └─［次曲なし］────────→ [待機中]（LED消灯／曲もOFF）
```
> ※ "曲終端で自動的に次曲再生。最後の曲なら停止して待機へ"

**各状態の意味：**

| 状態名 |内容|LED|パッシブブザー|LCDディスプレイ|
|:--|:--|:--|:--|:--|
|待機中|曲が鳴ってない。ボタン待ち|消灯|OFF|曲名表示|
|再生中|曲がなっている（選択中の曲 or 連続再生中）|点灯|曲再生中|曲名表示|

### 2-2. 機能一覧・関数一覧（役割の定義）

| 機能ID | 機能名 | 関数名 | 担う「1つの仕事」 | 主な引数 | 戻り値 | 呼び出す場所 |
|:--|:--|:--|:--|:--|:--|:--|
|—|初期化|setup()|ピン・LCD等の初期化|なし|なし|起動時|
|—|メインループ|loop()|各状態チェックと処理の分岐|なし|なし|常時|
|C01|ボタン入力読込|readButtons()|タクトスイッチのデバウンス・押下判定|なし|なし|loop()|
|C02|LED制御|updateLED()|再生中/停止中でLEDのON/OFF|なし|なし|loop()|
|C03|ブザー制御|playMelody()|指定曲（TrackID）の再生、状態切替でON/OFF|bool isPlay|なし|loop()|
|C04|LCD表示|updateLCD()|曲名や状態テキストのLCD表示|なし|なし|loop()|

|F03|自動次曲再生・最終曲で停止|playNextTrack()|1曲終了時にTrackID+1、曲数超過時に待機中遷移|なし|なし|playMelody()内から|

---

### 2-3. タイミング設計（delay vs millis）

| 処理 | 必要な周期 | delay / millis | 判断の理由 |
|:--|:--|:--|:--|
|LED点灯・消灯|状態変化時|millis|再生/停止に応じた即時反映でOK|
|ボタン入力監視（2つとも）|10～50ms|millis|入力ロスを防ぐため常時監視（チャタリング対策）|
|ブザー制御（メロディー再生）|音階ごと|delay|曲再生中のみ音階毎にdelayで間隔制御、他は並行可|
|LCD表示更新|画面切替時・曲変更時|millis|状態や曲切替に応じたタイミングで即時更新|
|曲終端処理（次曲判定）|曲終了毎|—|playNextTrack()|曲の終端判定と分岐処理|

## 4. 異常系・エラー処理の方針

- **TrackIDの範囲外（全曲再生後）は自動的に待機中に遷移、LED/ブザーを停止**
- 曲リスト範囲を超える参照や配列外アクセス防止、TrackCount変数でチェック

### Q1: 設計の妥当性確認

> 「この基本設計書をもとに Arduino UNO R3 でコードを書きます。ピン競合・ライブラリ競合・状態遷移の抜け・delay() の問題点はありますか？」

**AIの回答（要約）：**
- メロディーを再生するパートでdelay()を使うと、「再生中の他の入力（例：ボタン長押しで即停止）」が効かなくなります。
例: tone()で再生 ➔ delay(音階長さ)でループ →その間はボタン監視が停止
