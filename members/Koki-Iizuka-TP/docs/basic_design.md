## 1. システム全体像

### 1-1. ブロック構成（入力 → 処理 → 出力）

```
【入力】                                  【Arduino UNO R3】                 【出力】
タクトスイッチ①（再生/停止ボタン） ──→┐
                                         │
タクトスイッチ②（曲切替ボタン）   ───→┤    状態管理・判定処理    ├───→ LED
                                         │                            ├──→ パッシブブザー
                                         │                            └──→ LCDディスプレイ

補足説明（図で表現しきれない場合）
- 「タクトスイッチ①：曲の再生／停止を操作」
- 「タクトスイッチ②：再生中または待機中に曲を切り替える（複数曲対応時）」
- 曲再生時、自動で「次の曲」へ進み、すべて終了で自動停止・待機
- 最後の曲再生後はTrackID（選択曲番号）を0にリセット
```
---

### 1-2. 状態遷移（ガジェットの「ふるまい」設計）

```
[電源ON/初期化]
      ↓
  [待機中]  ←─────────────┐
   |  ↑                     |
   |  └──(再生/停止)        |
   |                       |
(再生/停止押下)             ↓
   ↓                [再生中]
  [再生中]
   |  ↑    ┌──────────────┐
   |  |    | 曲途中で停止  |
   |  └───(再生/停止押下)↓
   |                [待機中]
   |
   └▷（曲が終わる）
          ├─［次曲あり］─────────→ [再生中]（次の曲へ）
          └─［次曲なし］─────────→ [待機中]（LED消灯／曲もOFF／TrackIDリセット）
```

**各状態の意味：**

| 状態名 |内容|LED|パッシブブザー|LCDディスプレイ|
|:--|:--|:--|:--|:--|
|待機中|曲が鳴ってない。ボタン待ち|消灯|OFF|現在選択曲名表示（TrackID=0〜）|
|再生中|曲が鳴っている|点灯|曲再生中|再生中の曲名表示|

## 2. ソフトウェア設計

### 2-1. データ設計（使う変数・定数の洗い出し）

| 情報の名前 | 何を表すか | 型（イメージ） | バイト数 | 初期値 | 備考 |
|:--|:--|:--|:--|:--|:--|
|State|現在の状態（待機中／再生中など）|int|2B|0|状態遷移で使用|
|TrackID|現在選択している曲番号|int|2B|0|最終曲再生後に0へリセット|
|button1_State|タクトスイッチ①（再生/停止）の状態|bool|1B|false|デバウンス処理|
|button2_State|タクトスイッチ②（曲切替）の状態|bool|1B|false|デバウンス処理|
|Push_button1|SW①押下のタイミング|unsigned long|4B|0|チャタリング100ms|
|Push_button2|SW②押下のタイミング|unsigned long|4B|0|チャタリング100ms|
|led_State|LED点灯状態|bool|1B|false||
|buzzer_State|ブザー鳴動状態|bool|1B|false||
|TrackList[]|再生する曲データのリスト|int[][] または構造体配列|X|...|複数曲対応|
|Display_Text|LCDに表示する文字列|char[]|X|""|曲名や状態を表示（任意）|
|TrackCount|総曲数|int|2B|初期化時にセット|自動再生・終端判定用|

---

### 2-2. 機能一覧・関数一覧（役割の定義）

| 機能ID | 機能名 | 関数名 | 担う「1つの仕事」 | 主な引数 | 戻り値 | 呼び出す場所 |
|:--|:--|:--|:--|:--|:--|:--|
|—|初期化|setup()|ピン・LCD等の初期化|なし|なし|起動時|
|—|メインループ|loop()|各状態チェック／分岐|なし|なし|常時|
|C01|ボタン入力読込|readButtons()|タクトスイッチ（100msデバウンス）判定|なし|なし|loop()|
|C02|LED制御|updateLED()|状態に応じてON/OFF|なし|なし|loop()|
|C03|ブザー制御|playMelody()|現在TrackIDの曲を再生、ON/OFF制御|bool isPlay|なし|loop()|
|C04|LCD表示|updateLCD()|曲名や状態をLCD表示|なし|なし|loop()|
|F01|再生/停止切替|togglePlay()|再生↔停止の状態遷移|なし|なし|loop()|
|F02|曲切替|changeTrack()|TrackIDを+1し再生準備（最終曲なら0へ）|なし|なし|loop()|
|F03|自動次曲再生・最終曲で停止|playNextTrack()|曲終端時、TrackID+1して次曲再生。最終曲なら待機＆TrackIDリセット|なし|なし|playMelody()内|

---

### 2-3. タイミング設計（delay vs millis）

| 処理 | 必要な周期 | delay / millis | 判断の理由 |
|:--|:--|:--|:--|
|LED点灯・消灯|状態変化時|millis|即時反映|
|ボタン入力監視（２つ）|10〜100ms|millis|チャタリング100ms対策・常時監視|
|ブザー制御（メロディー再生）|音階ごと|delay|曲再生中のみ音階毎にdelay間隔で制御|
|LCD表示更新|画面切替時・曲変更時|millis|即時更新|
|曲終端処理|曲終了毎|—|playNextTrack()内で分岐処理|

## 4. 異常系・エラー処理の方針

| 異常ケース | 発生条件 | 検知方法 | 対応方針 |
|:--|:--|:--|:--|
|チャタリング|ボタン押し 100ms未満連打|前回入力から100ms未満|**デバウンス処理で無視**|
|LCD表示バグ|LCDに誤表示|文字列バッファ監視|直前の文字列再表示|
|曲データ不正|TrackID範囲外等|TrackCount判定|最初の曲（TrackID=0）に戻す|
