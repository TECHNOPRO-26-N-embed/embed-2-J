
# 基本設計書 — 組込み開発実習

<!-- 作成者: 加瀬 暖人 / 日付: 2026-05-26 / グループ: 2-j -->

## 1. 作品概要

### 1-1. タイトル
プッシュスイッチ連動型警報ガジェット

### 1-2. 一言説明
ボタン操作で異常を音で通知する装置

### 1-3. 動機・背景
異常発生時に利用者へ即時通知を行う仕組みは、防犯装置や設備監視など様々な場面で利用されている。
本作品では、Arduino Uno R3 のデジタル入力機能と出力制御機能を利用し、入力検知から通知までの基本的な組込み制御を学習することを目的とする。
また、シンプルな回路構成でリアルタイム制御を実現し、将来的なIoT機器開発の基礎知識習得につなげる。

## 2. ユーザーと利用シーン

### 2-1. 想定ユーザー
Arduino学習者、一般家庭利用者、オフィス利用者

### 2-2. 利用シーン
机の上や室内環境において、異常発生や操作入力を音で通知したい場面で使用する。
具体的には、防犯ブザー、簡易異常通知装置、ドア開閉通知などへの応用を想定する。

## 3. 機能要件

### 3-1. 必須機能
- プッシュスイッチの押下状態を検知できる
- ボタン押下時に受動ブザーから警告音を出力できる
- ボタン解除時にブザーを停止できる
- Arduino が常時入力状態を監視できる
- INPUT_PULLUP を利用して安定した入力判定ができる
- LED点灯による視覚通知ができる
- ブザー音程を変更できる
- 長押し時のみ警告音を鳴らせる
- 一定時間後に自動停止できる

### 3-2. スコープ外（やらないこと）
- Wi-Fi通信は実装しない
- スマートフォン連携は行わない
- クラウド接続は行わない
- 音声録音・再生機能は実装しない

### 3-3. 詳細動作条件
- ボタンが押された場合のみブザーを鳴動する
- ボタンが押されていない場合はブザーを停止する
- ブザー音は Arduino の tone() 関数を使用して出力する
- 出力周波数は 523Hz を基本とする
- INPUT_PULLUP を使用し、通常時は HIGH とする
- ボタン押下時に LOW 信号として判定する
- 屋内利用を前提とする
- 常温環境で利用する
- USB給電による動作を想定する

### 3-4. 非機能要件
| 項目     | 内容                                 |
|:--------|:-------------------------------------|
| 安全性   | 異常入力時はブザーを停止する         |
| 応答時間 | ボタン押下後1秒以内に反応する        |
| 安定性   | 誤動作なく連続動作できる             |
| 保守性   | 配線変更や部品交換を簡単に行える     |
| 拡張性   | LEDやセンサー追加が可能な構成とする  |

## 5. システム設計概要

### 5-1. ブロック図
```
【入力】        【Arduino UNO R3】       【出力】
プッシュスイッチ ─→ D2 ──┐                           ┌→ D3 ─→ 受動ブザー
                         │   判定・制御・監視まとめ  ─┤
                         └─────────────────────────┘
                                     │
                                     └→ D4 ─→ LED
```
補足：
- D2ピンはINPUT_PULLUPでボタン検知
- D3ピンでtone()利用・アクティブブザー出力
- D4はLED出力（必須機能）

### 5-2. 状態遷移（ふるまい設計）
```
[電源ON/初期化]
  ↓（初期化完了）
[待機]──（ボタン押下）──→[警報発報]
  ↑  └────────────（ボタン解除/タイムアウト/長押し終了）──┘
```
| 状態名      | この状態のとき何をしているか                                  | LED・ブザーの様子             |
|:-----------|:------------------------------------------------------------|:------------------------------|
| 初期化      | ピン初期化／変数リセット                                    | 消灯、ブザー停止              |
| 待機        | ボタン状態監視(チャタリング対策有)、入力待ち                 | 消灯、ブザー停止              |
| 警報発報    | ボタン押下中：ブザー鳴動・LED点灯                          | ブザーON、LED点灯             |
| 自動停止    | 長押し状態で一定時間後自動OFF                              | OFF                           |

## 4. 異常系・エラー処理の方針

| 異常ケース   | 発生条件             | 検知方法                     | 対応方針                         |
|:------------|:--------------------|:-----------------------------|:---------------------------------|
| チャタリング| ボタン押下時         | 前回変化時から50ms未満        | debounce(デバウンス)で入力無視   |
| ノイズ      | アナログ入力揺れ     | 閾値バッファ設定              | バッファ以内変化は保持           |
| ブザー異常   | tone/noTone効かない  | 実行後に出力状態確認          | 電気的故障時は再起動促進         |
| LED断線     | LED点かない          | 実出力確認                    | 部品交換等物理修理               |
