要件定義書
Arduino Uno R3 を用いた警報通知システム
1. 文書概要
1.1 目的

本書は、Arduino Uno R3 を用いた警報通知システムの要件を定義するものである。

1.2 システム概要

プッシュスイッチ入力を検知し、異常発生時に受動ブザーを用いて警告音を出力するシステムを構築する。

2. システム構成
2.1 使用機器
機器名	用途
Arduino Uno R3	制御装置
プッシュスイッチ	入力装置
受動ブザー	音声出力装置
ブレッドボード	回路構築
ジャンパーワイヤ	配線
抵抗（220Ω）	電流制御
3. 機能要件
3.1 入力機能
プッシュスイッチ押下をデジタル入力として取得すること。
スイッチ状態を常時監視すること。
3.2 出力機能
スイッチ押下時にブザーを鳴動させること。
スイッチ解除時にブザーを停止すること。
3.3 制御機能
Arduino によりリアルタイム制御を行うこと。
tone() 関数を利用して警告音を出力すること。
4. 非機能要件
項目	内容
安定性	誤動作が発生しないこと
保守性	配線・プログラムが簡易であること
拡張性	センサー追加が可能であること
応答性	ボタン押下後すぐに反応すること
5. 動作仕様
状態	システム動作
通常時	ブザー停止
ボタン押下時	ブザー鳴動
ボタン解除時	ブザー停止
6. 工夫した点
6.1 INPUT_PULLUP の活用

Arduino 内部プルアップ抵抗を利用することで、外部回路を簡略化し、ノイズによる誤検知を低減した。

6.2 受動ブザーの採用

単純なON/OFF制御ではなく、tone() 関数による周波数制御を可能にし、将来的な警告音パターン拡張に対応できる構成とした。

6.3 リアルタイム監視

loop() 関数による継続監視を行うことで、入力に対して即時反応するシステムを実現した。

7. 制約条件
Arduino Uno R3 を使用すること。
デジタル入力機能を利用すること。
出力装置は受動ブザーを使用すること。
8. 将来的な拡張性・解決可能課題

本システムを拡張することで、以下の課題解決が可能となる。

防犯アラームシステム
高齢者向け緊急通知装置
工場設備異常通知
ドア開閉監視
IoT監視システム

さらに、Wi-Fi モジュールや人感センサーを追加することで、遠隔監視システムへの発展が可能である。

9. 完成条件

以下を満たした場合、本システムは完成とする。

スイッチ押下時にブザーが鳴動する
スイッチ解除時に停止する
安定して連続動作する
誤動作が発生しない
