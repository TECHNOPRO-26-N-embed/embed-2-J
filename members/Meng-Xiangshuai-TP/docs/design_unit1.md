# 基本設計書 — RFID認証警告システム

## 0. 要件定義書との接続確認

| 項目 | 内容 |
|:--|:--|
| 作品タイトル | RFID認証警告システム |
| 一言説明 | スイッチを押した時だけRFIDを読み取り、未登録カードならブザーで警告し、認証成功時はLEDを点灯する |
| 使用する共通部品 | スイッチ |
| 追加部品 | RC522 RFIDモジュール、ブザー、LED |
| 必須機能の数 | 4件 |
| 追加機能の数 | 3件 |

## 0-2. グループ共通部品との接続設計

| 項目 | 内容 |
|:--|:--|
| グループ共通部品 | スイッチ |
| 共通部品が担う役割 | RFID認証を開始するための入力 |
| 接続するArduinoピン | D2 |
| 個人の設計と共通部品が協調する箇所 | スイッチが押された時のみRFIDカードを読み取る |
| グループ内で統一すべき仕様 | INPUT_PULLUPを使用し、押下時はLOWとして扱う |

## 1. システム全体像

### 1-1. ブロック構成

```text
【入力】                    【Arduino UNO R3】                  【出力】
       スイッチ ────→┐                         ┌──→ LED（緑）
  RFIDカードー ─────→┤   状態管理・判定処理   (認証成功) 
                      └─────────────────────────┘
                          RFID読み取り処理   |　　(認証失敗)
                                            └─────→  ブザー

``` 
### 1-2. 状態遷移（ガジェットの「ふるまい」設計）

[電源ON / 初期化]
  ↓
[待機中] ──（スイッチ押下）──→(カードをタッチ)──→ [RFID読取中]
  ↓                              　　　　　　　　　　↓
[待機中] ←──(LED)←─（登録済みUID）←── (UID確認)　←──　[認証成功]
  ↓                                ↓
[待機中] ←─ [警告(ブザー)]　←──（未登録UID）


**各状態の意味：**
| 状態名     | 内容             | 出力          |
| :------ | :------------- | :---------- |
| 待機中     | スイッチ入力を待つ      | LED消灯、ブザー停止 |
| RFID読取中 | カードのUIDを読み取る   | 出力なし        |
| 認証成功    | 登録済みUIDと一致した状態 | LED点灯       |
| 警告中     | 未登録UIDを検知した状態  | ブザーを長く鳴らす   |


## 2. ソフトウェア設計

### 2-1. データ設計（使う変数・定数の洗い出し）


| 情報の名前         | 何を表すか       | 型         | 初期値    |
| :------------ | :---------- | :-------- | :---------- |
| SW_PIN        | スイッチのピン番号   | const int 　　　　|　 2  |
| LED_PIN       | LEDのピン番号    　　| const int 　　　　|　5  |
| BUZZER_PIN    | ブザーのピン番号    　| const int 　　　　| 8   |
| SS_PIN        | RFIDのSDAピン  　　　| const int 　　　　| 10   |
| RST_PIN       | RFIDのRSTピン  　　　| const int 　　　　| 9    |
| registeredUID | 登録済みカードのUID 　| byte配列    　　　| E9 EF 1A 07 |
| currentState  | 現在の状態       　　　| int      　　　 | 0    |


### 2-2. 機能一覧・関数一覧（役割の定義）

| 機能ID | 機能名    | 関数名             | 役割                |
| :--- | :----- | :-------------- | :---------------- |
| —    | 初期化    | setup()         | ピン設定、SPI、RFID初期化  |
| —    | メイン制御  | loop()          | 状態に応じて処理を呼び出す     |
| F01  | スイッチ入力 | readSwitch()    | スイッチが押されたか確認する    |
| F02  | RFID読取 | readRFID()      | RFIDカードのUIDを取得する  |
| F03  | UID判定  | checkUID()      | 登録済みUIDと一致するか判定する |
| F04  | LED通知  | successLED()    | 認証成功時にLEDを点灯する    |
| F05  | ブザー警告  | warningBuzzer() | 未登録カード時にブザーを鳴らす   |


### 2-3. タイミング設計（delay vs millis）

| 処理 | 必要な周期 | delay  | 判断の理由 |
|:--|:--|:--|:--|
| RFIDカード読み取り | スイッチ押下時 | delay | 認証処理を順番に行うため |
| ブザー警告 | 1000ms | delay | 一定時間ブザーを鳴らすだけのため |
| LED点灯 | 認証成功時 | delay | 単純なON/OFF制御のため |

## 3. ハードウェア設計

### 3-1. 部品一覧とピン接続

| 部品名        | 種類       | Arduinoピン | 入力/出力 | 備考             |
| :--------- | :------- | :-------- | :---- | :------------- |
| スイッチ　| タクトスイッチ| D2        | 入力    | INPUT_PULLUP使用 |
| LED      　| 5mm LED  | D5        | 出力    | 220Ω抵抗と直列      |
| ブザー | アクティブブザー | D8        | 出力    | 認証失敗時に鳴動       |
| RC522 SDA  | RFID     | D10       | 入力    | SPI通信          |
| RC522 MOSI | RFID     | D11       | 入力    | SPI通信          |
| RC522 MISO | RFID     | D12       | 出力    | SPI通信          |
| RC522 SCK  | RFID     | D13       | 入力    | SPI通信          |
| RC522 RST  | RFID     | D9        | 入力    | リセット用          |
| RC522 VCC  | RFID     | 3.3V      | 電源    | 5V不可           |
| RC522 GND  | RFID     | GND       | 電源    | 共通GND          |


### 3-2. 電気的考慮事項

> **【任意】** 余裕があれば記入してください。抵抗値はAIに計算させてOKです。

| 用途 | 判断・計算メモ |
|:--|:--|
| LED電流制限抵抗 | (5V − 2.0V) ÷ 0.02A ≒ 150Ω のため、220Ω抵抗を使用 |
| ボタンのプルアップ | INPUT_PULLUPを使用するため外付け抵抗は不要 |
| RFIDモジュールの動作電圧 | RC522は3.3V動作のため、Arduinoの3.3V端子へ接続する |
| ブザーの動作 | デジタル出力によるON/OFF制御を行う |
| RFID通信 | RC522はSPI通信を使用するため、D10〜D13ピンを使用する |

## 4. 異常系・エラー処理の方針
| 異常ケース      | 発生条件              | 対応方針                      |
| :--------- | :---------------- | :------------------------ |
| チャタリング     | スイッチ押下時           | 50ms程度待機して安定化する           |
| RFID読み取り失敗 | カードが正しく読めない       | 再度カードをかざす                 |
| 未登録カード     | UIDが登録済みUIDと一致しない | ブザーを長く鳴らす                 |
| 配線ミス       | RFIDが反応しない        | Serial Monitorで確認し、配線を見直す |

## 7. AIレビュー記録
Q1: 設計の妥当性確認

AIの回答：
ピン競合はなく、入力・処理・出力の流れも明確である。

対応した内容：
SPI通信で使用するD10〜D13を避けて、スイッチ・LED・ブザーのピンを設定した。

Q2: チャタリング・delayの確認

AIの回答：
スイッチにはチャタリング対策が必要である。

対応した内容：
スイッチ押下後に短い待機時間を入れる方針とした。
