## 1. システム全体像

### 1-1. ブロック構成（入力 → 処理 → 出力）

```
【入力】              【Arduino UNO】              【出力】
スイッチボタン  →   制御・乱数生成   →   7セグメントLED（1〜6）
                                      →   ブザー音

```

### 補足説明（図で表現しきれない場合）
- 起動時に1〜6の数字を順番に表示し、ブザー音を鳴らす演出があります。
- ボタンを長押ししても、1回だけサイコロが振られます。
- ブザーは元の要件には含まれていませんが、追加機能として実装されています。
- アニメーションやチャタリング対策にはdelay()ではなくmillis()を使い、他の処理を止めないようにしています。

---

### 1-2. 状態遷移（ガジェットの「ふるまい」設計）


```

```
[待機状態]
   ↓（ボタン押下）
[アニメーション（roll）]
   ↓
[結果表示]
   ↓
[待機状態へ戻る]


**各状態の意味：**

| 状態名 | この状態のとき、何をしているか | LEDやブザーなど出力の様子 |
|:--|:--|:--|
| （例）待機中 | センサー値を100msごとに監視する | 緑LED：500msで点滅 |
| 待機 | ボタン待ち | 前回の数字 |
|回転  | ランダムに高速変化 | 点滅表示 |
| 結果 | 1〜6を表示 | 固定表示 |
| 初期化 | 起動処理 | カウントダウン |
- ボタンを長押ししても、1回だけサイコロが振られます。

### 2-2. 機能一覧・関数一覧（役割の定義）


| 機能   | 関数名           | 内容         |
| :--- | :------------ | :--------- |
| 初期化  | setup()       | ピン設定・起動処理  |
| メイン  | loop()        | ボタン監視      |

### 2-3. タイミング設計（delay vs millis）


| 処理      | 方法          | 理由    |
| :------ | :---------- | :---- |
| ボタン     | digitalRead + millis | 即時反応・ノンブロッキング  |
| アニメーション | millis       | 見た目重視・ノンブロッキング |
| 起動演出    | delay       | 視覚効果  |

## 4. 異常系・エラー処理の方針

| 問題     | 対策        |
| :----- | :-------- |
| チャタリング | millis()を使い、ボタンを押した後、離すまで待機し、さらに200msのディレイ相当時間でチャタリングを防止しています。|
| ノイズ不足  | A5ピンを乱数シード用ノイズ取得専用として使用しています。これにより乱数の安定性を確保しています。 |
